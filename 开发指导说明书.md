# AI 自动循环系统 - 开发指导说明书

> 本文档面向由 `run.sh` 调用的 AI（Claude Code CLI），帮助其理解任务上下文、文件格式和工作约束。

---

## 第一部分：如何编写 task.json

### 完整 Schema

```json
{
  "version": "1.0.0",
  "config": {
    "max_retries": 2,
    "timeout_seconds": 300,
    "max_budget_usd": 1.0
  },
  "tasks": [
    {
      "id": "task-001",
      "title": "任务简述",
      "prompt": "给 Claude 的完整指令"
    }
  ]
}
```

### 字段说明

#### config（全局配置）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `max_retries` | integer | 2 | 单任务最大重试次数（含首次执行） |
| `timeout_seconds` | integer | 300 | 单次执行超时（秒） |
| `max_budget_usd` | number | 1.0 | 单次执行最大费用（美元） |

#### tasks（任务数组）

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | string | 是 | 唯一，建议格式 `task-NNN` | 任务唯一标识 |
| `title` | string | 是 | - | 简短描述（用于日志和报告） |
| `prompt` | string | 是 | - | 发送给 Claude 的完整指令 |

### Prompt 编写最佳实践

**好的 prompt：**
```
在 projects/hello-world/ 目录下创建一个 Node.js 项目：
1. 创建 package.json（name: hello-world, version: 1.0.0）
2. 创建 index.js，导出一个函数 greet(name)，返回 "Hello, {name}!"
3. 创建 index.test.js，使用 Node.js 内置 test runner 测试 greet 函数
4. 验收标准：运行 node --test 全部通过
```

**差的 prompt：**
```
写个 hello world
```

### 关键规则

- **所有项目代码必须在 `projects/<project-name>/` 子目录下开发**
- prompt 中应明确指定项目路径，如 `projects/my-app/`
- Claude 的工作目录（cwd）是 `projects/`，所以 prompt 中路径相对于 `projects/`
- 每个任务应独立完整，不依赖其他任务的执行结果
- prompt 应包含明确的验收标准

## 第二部分：如何记录 report.json

### 完整 Schema

report.json 是一个 **纯数组**，每次任务执行（包括重试）追加一条记录：

```json
[
  {
    "task_id": "task-001",
    "task_title": "任务简述",
    "status": "success",
    "attempt": 1,
    "started_at": "2026-03-12T15:00:00+08:00",
    "finished_at": "2026-03-12T15:00:42+08:00",
    "duration_seconds": 42,
    "exit_code": 0,
    "error_message": null
  }
]
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | string | 对应 task.json 中的 id |
| `task_title` | string | 对应 task.json 中的 title |
| `status` | string | 枚举值，见下表 |
| `attempt` | integer | 当前是第几次尝试（从 1 开始） |
| `started_at` | string | ISO 8601 格式开始时间 |
| `finished_at` | string | ISO 8601 格式结束时间 |
| `duration_seconds` | integer | 执行耗时（秒） |
| `exit_code` | integer | Claude CLI 退出码 |
| `error_message` | string \| null | 错误信息（成功时为 null） |

### status 枚举值

| 值 | 含义 | exit_code |
|----|------|-----------|
| `success` | 任务成功完成 | 0 |
| `failure` | 任务执行失败 | 非 0 且非 124 |
| `timeout` | 执行超时被终止 | 124 |
| `skipped` | 任务被跳过（预留） | - |

### 中断恢复逻辑

run.sh 通过以下逻辑判断任务是否已完成：

1. 读取 report.json 中所有 `status === "success"` 的记录
2. 提取这些记录的 `task_id`，组成 "已完成集合"
3. 遍历 task.json 的 tasks 数组，跳过 "已完成集合" 中的任务
4. 找到第一个未完成的任务开始执行

因此：
- 只有 `status: "success"` 算完成，failure/timeout 不算
- 同一个 task_id 可以有多条记录（重试产生的）
- 中断后重启，会自动从上次未完成的任务继续

## 第三部分：run.sh 工作机制

### 执行流程

```
1. 初始化检查（jq、claude、task.json）
2. 读取全局 config
3. 主循环：
   a. 从 report.json 获取已完成的 task_id 列表
   b. 从 task.json 找第一个未完成的 task
   c. 如果全部完成 → 输出摘要，退出
   d. 执行任务（带重试循环）
   e. 回到 step a
4. 输出最终统计摘要
```

### Claude CLI 执行参数

```bash
timeout $TIMEOUT claude -p "$(cat prompt.tmp)" \
  --output-format json \
  --dangerously-skip-permissions \
  --max-budget-usd $BUDGET \
  --no-session-persistence \
  > logs/task-xxx_attemptN.json 2>&1
```

| 参数 | 说明 |
|------|------|
| `-p "..."` | 以 print 模式运行，传入 prompt |
| `--output-format json` | JSON 格式输出 |
| `--dangerously-skip-permissions` | 跳过权限确认，全自动执行 |
| `--max-budget-usd` | 限制单次调用费用 |
| `--no-session-persistence` | 不保留会话状态 |

### 错误处理

- **exit_code 0**: 成功，记录 `success`
- **exit_code 124**: 超时（由 `timeout` 命令产生），记录 `timeout`
- **其他 exit_code**: 失败，记录 `failure`，等待 5 秒后重试

### 日志说明

- `logs/run_YYYYMMDD_HHMMSS.log`: 本次运行的完整控制台日志
- `logs/task-xxx_attemptN_YYYYMMDD_HHMMSS.json`: 每次 Claude 执行的原始输出
- 控制台和日志文件同步输出（通过 tee 实现）
